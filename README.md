# E-Commerce Big Data Analytics

Analyzing 2.6M+ transaction records from an online electronics store to uncover sales trends, customer behavior, and category/brand performance — with a full preprocessing pipeline, dimensionality reduction, and customer segmentation.

## Overview

This project applies a complete big data analytics workflow to a large, messy, real-world e-commerce transaction log. Raw transaction data is rarely clean — this one included duplicate records, corrupted placeholder timestamps, and heavy missingness in customer and pricing fields. The project builds a preprocessing pipeline to systematically handle these issues, engineers customer-level behavioral features, applies PCA for dimensionality reduction, and produces a set of visualizations that translate directly into business decisions around inventory, marketing, and pricing.

## Dataset

- **Source:** [eCommerce Purchase History from Electronics Store](https://www.kaggle.com/datasets/mkechinov/ecommerce-purchase-history-from-electronics-store) (Kaggle)
- **Size:** 2,633,521 rows × 8 columns
- **Fields:** `event_time`, `order_id`, `product_id`, `category_id`, `category_code`, `brand`, `price`, `user_id`
- **Period covered:** January – November 2020

## What this project does

1. **Data cleaning** — removes duplicates, drops corrupted placeholder (`1970-01-01`) timestamps, drops incomplete records missing `user_id`/`price`, fills missing categorical fields, removes price outliers via IQR, and collapses long-tail rare brands into an "Other" bucket
2. **Feature engineering** — derives monthly/daily/hourly purchase patterns and per-customer aggregates (total spend, average spend, purchase frequency, max price)
3. **Exploratory & analytical visualization** — monthly sales trend, top categories, top brands, price distribution, and spending-vs-frequency analysis
4. **Dimensionality reduction** — standardizes and compresses 4 correlated customer features into 2 principal components via PCA, retaining 94.5% of variance, for customer segmentation
5. **Exports a cleaned dataset** ready for further modeling (e.g., clustering, churn prediction)

## Results

| Metric                                 | Value                      |
| -------------------------------------- | -------------------------- |
| Raw rows                               | 2,633,521                  |
| Duplicate rows removed                 | 675                        |
| Invalid (epoch) timestamp rows removed | 19,631                     |
| Rows missing user_id/price removed     | 2,051,027                  |
| Price outliers removed (IQR)           | 43,596                     |
| **Final clean rows**                   | **518,553**                |
| Unique customers                       | 91,768                     |
| Unique products                        | 19,397                     |
| PCA variance retained (2 components)   | 94.5%                      |
| Top brand                              | Samsung (86,479 purchases) |

**Key findings:**

- Sales show a clear, non-uniform monthly trend with a strong peak in August
- Purchases are concentrated in a small number of categories and brands — Samsung alone accounts for a disproportionate share of volume
- Most transactions cluster at the lower end of the price range, with a clear long right tail
- A subset of high-frequency customers accounts for a disproportionately large share of total revenue, making them strong loyalty/retention targets
- Customer purchasing behavior is well captured by just 2 latent dimensions (spend level and purchase frequency), validating PCA-based segmentation

## Getting Started

### Prerequisites

```bash
pip install pandas numpy matplotlib seaborn scikit-learn
```

### Setup

1. Download the dataset from [Kaggle](https://www.kaggle.com/datasets/mkechinov/ecommerce-purchase-history-from-electronics-store) and save it as `data/kz.csv`
2. Open `notebook/ecommerce_analysis.ipynb`
3. Run all cells top to bottom

The notebook will print row counts removed at each cleaning step, save all figures to `images/`, and export the cleaned dataset to `data/kz_cleaned.csv`.
